# MyMaps — Android-приложение для городских маршрутов

MyMaps — мобильное приложение для поиска, сохранения и создания городских маршрутов.
Сервис помогает пользователям находить интересные места для прогулок, поездок и активного отдыха, а также делиться собственными маршрутами с фотографиями, описанием и точками на карте.

Проект разрабатывался как полноценное Android-приложение с акцентом на современную архитектуру, чистую навигацию и масштабируемость.

---

## Что умеет приложение

### 🔍 Поиск маршрутов
Пользователь может просматривать ленту маршрутов и быстро находить подходящие варианты:

- поиск по названию;
- фильтрация по расстоянию, длительности, рейтингу и количеству точек;
- подбор маршрутов под сценарии использования — прогулка, фотосессия, свидание, бары, развлечения и т.д.;
- сортировка по рейтингу, расстоянию и времени прохождения.

Маршруты отображаются в виде карточек с изображением, рейтингом и краткой информацией.

---

### 🗺 Карта маршрутов
В приложении реализован отдельный экран карты на базе Yandex MapKit:

- отображение всех маршрутов на карте;
- кластеризация маркеров;
- открытие карточки маршрута по нажатию на точку;
- поддержка светлой и тёмной темы;
- геолокация пользователя.

---

### ❤️ Избранное
Понравившиеся маршруты можно сохранять в избранное и быстро возвращаться к ним позже.

Состояние избранного хранится локально через Room.

---

### ✍️ Конструктор маршрутов
Одна из ключевых частей проекта — полноценный flow создания собственного маршрута.

Пользователь может:

- загрузить обложку маршрута;
- указать название и описание;
- добавить точки маршрута;
- выбрать координаты на карте;
- прикрепить фотографии;
- назначить теги и категории.

Все этапы редактора объединены единым Shared ViewModel, благодаря чему состояние сохраняется между экранами без ручной передачи данных.

---

### 👤 Авторизация
В приложении реализованы:

- вход и регистрация;
- восстановление пароля;
- авторизация через VK ID One Tap;
- Splash-screen с автоматической проверкой токена.

## Архитектурный подход

### MVI / UDF
Каждый экран построен по принципу:

- `State` — единственный источник состояния UI;
- `Event` — действия пользователя;
- `Effect` — одноразовые события (навигация, snackbar и т.д.).

Такой подход делает UI предсказуемым, упрощает тестирование и снижает связанность между слоями.

---

### Type-safe навигация
Навигация построена через типизированные маршруты:

```kotlin
@Serializable
data class RouteInfo(val routeId: Long)
```

Это позволяет безопасно передавать аргументы между экранами без строковых route.

---

### Модульная feature-структура
Проект разделён по feature-модулям:

```text
features/
├── auth/
├── explore/
├── favourite/
├── editor/
├── map/
├── profile/
└── navigation/
```

Каждая feature содержит собственные:

- UI-экраны;
- ViewModel;
- state/event/effect;
- data/domain-слои.

Это упрощает поддержку и дальнейшее масштабирование приложения.

---

### Работа с состоянием
Навигация не передаётся напрямую в UI.

Экран отправляет события во ViewModel:

```kotlin
onEvent(RouteInfoEvent.BackClicked)
```

ViewModel генерирует `Effect`, который обрабатывается на уровне NavHost.

Так UI остаётся независимым от `NavController`.

## Сборка проекта

### Требования

- Android Studio Ladybug или новее
- JDK 11+
- Android SDK 36

---

### Настройка

1. Клонируйте репозиторий:

```bash
git clone https://github.com/evseenkovia/vk-android-vkat.git
```

2. Добавьте API-ключ Yandex MapKit в `local.properties`:

```properties
MAPKIT_API_KEY=YOUR_API_KEY
```

3. Синхронизируйте проект с Gradle и запустите на устройстве или эмуляторе.

> VK ID уже настроен в `build.gradle.kts` — при необходимости замените `VKIDClientID` и `VKIDClientSecret` на ключи своего приложения из [VK для разработчиков](https://dev.vk.com/).
